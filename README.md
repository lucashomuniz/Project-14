# ✅ PROJECT 14

Este projeto tem como objetivo abordar o processo de Machine Learning no contexto de um problema na área da Engenharia Civil, especificamente relacionado ao concreto. Em vez de realizar as tarefas manualmente, iremos criar módulos de automação para desenvolver nosso próprio sistema de AutoML, sem depender de frameworks específicos. Utilizaremos o Spark MLlib no PySpark para aplicar técnicas de Machine Learning. No campo da Engenharia Civil, o concreto é um dos materiais mais importantes. A resistência à compressão do concreto é uma propriedade essencial, que depende de forma não linear da idade do concreto e dos ingredientes utilizados em sua composição. Neste projeto, nosso objetivo será construir um modelo preditivo capaz de estimar a resistência característica à compressão do concreto.Utilizaremos um dataframe de dados disponível publicamente, no qual a variável alvo é a "Concrete Compressive Strength" (resistência à compressão do concreto em MPa) e as demais variáveis serão usadas como preditoras.

Dado que estamos prevendo um valor numérico e temos dados de entrada e saída, trata-se de um problema de regressão. Para encontrar o melhor modelo preditivo, experimentaremos diferentes algoritmos de regressão, como Linear Regression, DecisionTreeRegressor, RandomForestRegressor, GBTRegressor e IsotonicRegression. Utilizaremos técnicas de otimização e ajuste de hiperparâmetros para alcançar o melhor desempenho possível. Após o treinamento do modelo, faremos previsões utilizando novos dados. Isso nos permitirá avaliar a capacidade de generalização do modelo e sua aplicabilidade em situações reais. Ao desenvolver nosso próprio sistema de AutoML, teremos maior controle sobre o processo de Machine Learning e poderemos explorar diferentes algoritmos e técnicas para obter os melhores resultados. Com base na análise comparativa de desempenho dos algoritmos de regressão utilizados, selecionaremos aquele que apresentar a melhor performance para nosso problema específico de previsão da resistência à compressão do concreto.

# ✅ PROCESS

O MLlibe, também conhecido como MLlib (Machine Learning Library), é uma biblioteca poderosa e versátil de machine learning projetada para ser usada com o Apache Spark, uma plataforma de processamento distribuído e análise de big data. Com o MLlibe, é possível realizar tarefas complexas de machine learning em larga escala, aproveitando a capacidade de processamento distribuído do Spark. No MLlibe, a preparação adequada dos dados é um passo crucial antes de treinar modelos de machine learning. Uma das exigências essenciais do MLlibe é que todas as colunas de entrada do dataframe sejam vetorizadas. Isso significa que as características presentes nessas colunas devem ser transformadas em representações numéricas, na forma de vetores, para que possam ser processadas pelos algoritmos de machine learning disponíveis no MLlibe. A vetorização das colunas de entrada é uma etapa importante, pois permite que os algoritmos do MLlibe trabalhem com os dados de maneira eficiente e precisa. A vetorização envolve a transformação das características de cada coluna em um vetor numérico, que representa de forma quantitativa as informações contidas nessas características. Dessa forma, o MLlibe pode realizar cálculos matemáticos e estatísticos nos vetores para treinar e avaliar os modelos de machine learning.

AutoML (Automatic Machine Learning) é uma abordagem que visa automatizar o processo de construção de modelos de machine learning. Tradicionalmente, o desenvolvimento de modelos de machine learning envolve várias etapas, como a seleção e preparação dos dados, a escolha e ajuste de algoritmos, a otimização de hiperparâmetros e a avaliação dos modelos. O AutoML busca simplificar e agilizar esse processo, permitindo que usuários com diferentes níveis de experiência possam aproveitar os benefícios do machine learning sem necessariamente se tornarem especialistas em ciência de dados. O objetivo do AutoML é fornecer ferramentas e técnicas automatizadas que possam realizar várias etapas do processo de construção do modelo, reduzindo a necessidade de intervenção manual. Isso inclui a automação de tarefas como a pré-processamento de dados, seleção de algoritmos, ajuste de hiperparâmetros e avaliação do modelo.

Existem várias ferramentas e plataformas de AutoML disponíveis, que oferecem uma variedade de recursos. Essas ferramentas podem utilizar abordagens como busca de hiperparâmetros automatizada, seleção de modelos automatizada e até mesmo geração automática de código. Além disso, muitas ferramentas de AutoML fornecem recursos para lidar com desafios comuns, como a detecção e tratamento de dados ausentes, balanceamento de classes desequilibradas e interpretabilidade do modelo. Embora o AutoML seja útil e eficiente em muitos casos, é importante ter em mente que ele não substitui a experiência e o conhecimento humano. Em alguns cenários complexos ou exclusivos, pode ser necessário um ajuste manual mais detalhado ou personalizado. No entanto, o AutoML desempenha um papel importante ao democratizar o acesso ao machine learning, permitindo que uma variedade de profissionais possam explorar e utilizar essa tecnologia em seus projetos, mesmo sem ter um conhecimento profundo de ciência de dados.

# ✅ CONCLUSION

Este projeto utiliza o MLlibe, uma biblioteca de machine learning desenvolvida para o Apache Spark, para realizar tarefas de treinamento de modelos de machine learning. Antes de treinar os modelos, é necessário preparar os dados de acordo com as exigências do MLlibe. Uma das exigências fundamentais do MLlibe é que todas as colunas de entrada do dataframe sejam vetorizadas, ou seja, que as características presentes nessas colunas sejam convertidas em vetores numéricos. Isso é necessário para que os algoritmos de machine learning possam processar corretamente os dados. No código fornecido, são realizadas várias etapas para preparar os dados antes de treinar os modelos. Primeiro, é feita a leitura do arquivo de dados usando o Spark e exibindo algumas informações básicas sobre o dataframe, como o número de registros e a visualização dos dados.

Em seguida, é definida uma função chamada `func_modulo_prep_dados` que automatiza a preparação dos dados. Essa função trata valores ausentes, converte variáveis categóricas em numéricas, lida com outliers, realiza vetorização das colunas de entrada e padroniza os dados, se necessário. Ao final, a função retorna um novo dataframe com as colunas preparadas. Após a preparação dos dados, é realizada a verificação da correlação entre as variáveis do dataframe usando a função `Correlation.corr`. Isso permite identificar possíveis multicolinearidades que podem afetar o desempenho dos modelos de machine learning. Em seguida, é criada uma função chamada `func_modulo_ml` que automatiza o uso de vários algoritmos de regressão. Essa função cria, treina e avalia cada modelo usando diferentes combinações de hiperparâmetros. O objetivo é identificar o modelo com melhor desempenho.

A função `func_modulo_ml` é aplicada a uma lista de algoritmos de regressão, como Linear Regression, Decision Tree Regression, Random Forest Regression, GBT Regression e Isotonic Regression. Para cada algoritmo, o código treina o modelo usando validação cruzada e avalia o desempenho do modelo com base nas métricas de erro RMSE (Root Mean Squared Error) e coeficiente R2. Ao final do treinamento dos modelos, é exibida uma tabela resumindo o desempenho de cada algoritmo. Com base nesses resultados, o modelo Gradient-Boosted Tree (GBT) é selecionado como o de melhor desempenho e será usado em produção. O código também inclui um exemplo de como fazer previsões com o modelo treinado. São fornecidos dados de teste e o modelo GBT é utilizado para fazer as previsões com esses dados. Em resumo, esse projeto utiliza o MLlibe e o Spark para preparar dados, treinar modelos de regressão e fazer previsões. Automatiza várias etapas do processo e permite escolher o melhor modelo com base no desempenho.
